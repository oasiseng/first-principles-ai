# Changelog

All notable changes to this project are documented here.

## [1.0.0] - 2026-02-16
### Added
- Added a vendor-neutral seven-principle core in `PRINCIPLES.md`.
- Added machine-readable principle definitions in `principles.yaml` and validation rules in `schema.json`.
- Added implementation resources: `IMPLEMENTATION_GUIDE.md`, `CHECKLIST.md`, `/templates`, `/guides`, and `/mappings`.
- Added governance and adoption artifacts: `GOVERNANCE.md`, `CODE_OF_CONDUCT.md`, `CITATION.cff`, and GitHub self-assessment issue template.
- Added `examples/incidents.md` as a "what breaks" companion for principle failure modes.
### Changed
- Rewrote `README.md` into a concise, adoption-focused quick-start with principle summary, navigation, and citation block.
- Updated `CONTRIBUTING.md` to a lightweight RFC-driven process with 14-day review windows.


## [0.1.1] - 2026-02-10
### Changed
- Updated Commandments I-II in `CONSTITUTION.md` to: "AI Shall Support Humans Above All Things" and "AI Shall Never Lie and Must Always Seek Truth".
- Shifted previous commandments to III-IX and aligned `RATIONALE.md` and `SCENARIOS.md` numbering and content.

## [0.1.0] - 2026-02-10
### Added
- Initial project scaffolding and mission-driven README.
- Draft `CONSTITUTION.md` with Part I Commandments and placeholders for Parts II-IV.
- Supporting docs: `RATIONALE.md`, `SCENARIOS.md`, `GLOSSARY.md`, `CONTRIBUTING.md`.
- `discussions/` and `translations/` directories.
